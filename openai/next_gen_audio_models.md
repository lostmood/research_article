# Next-Generation Audio Models in the API

## 基本信息
- **来源**: OpenAI Blog / API
- **发布日期**: 2025-03-20
- **链接**: https://openai.com/index/next-gen-audio-models/
- **类别**: API 模型发布 / 语音技术 (Speech Technology)

## 核心概述

OpenAI 发布了新一代音频模型系列，包括文本转语音 (TTS) 的 **gpt-4o-mini-tts** 和语音转文本 (STT) 的 **GPT-4o Transcribe / GPT-4o Mini Transcribe**。这些模型全面替代了此前的 TTS 和 Whisper 系列模型。gpt-4o-mini-tts 的核心创新在于**可提示的语音风格控制** (Steerable Voice)，开发者可以通过自然语言指令定制语音的情感和风格；Transcribe 系列则在多语言准确率上实现了对 Whisper 的全面超越，尤其在嘈杂环境和非英语语言上表现突出。

## 关键技术细节

### 一、gpt-4o-mini-tts（文本转语音）

#### 模型版本
| 模型标识 | 发布时间 |
|---------|---------|
| `gpt-4o-mini-tts` | 别名（指向最新版本） |
| `gpt-4o-mini-tts-2025-03-20` | 初始快照版本 |
| `gpt-4o-mini-tts-2025-12-15` | 更新快照版本 |

#### 核心特性：可提示语音控制 (Steerable Voice)

基于 GPT-4o Mini 语言模型构建，**通过自然语言 prompt 控制语音风格和情感**：
- **客服场景**：可设置同理心、耐心的语调
- **儿童故事**：可设置活泼、富有幻想感的语调
- **商务场景**：可设置专业、正式的语调
- 开发者可同时定制语音的"体验" (experience) 和"上下文" (context)

#### 可用语音 (Voices)
提供 **13+ 种预设语音**：Alloy、Ash、Ballad、Coral、Echo、Fable、Nova、Onyx、Sage、Shimmer、Verse 等

#### 输出格式
支持 MP3、Opus、AAC、FLAC、WAV、PCM 等多种格式

#### 定价

| 组件 | 价格 |
|------|------|
| **文本输入 Tokens** | $0.60 / 1M tokens |
| **音频输出 Tokens** | $12.00 / 1M tokens |
| **大约每分钟成本** | ~$0.015/min |

相比旧版 TTS Standard ($15/1M 字符) 和 TTS HD ($30/1M 字符)，**成本效益更高**。

#### 限制
- 不支持视觉输入和函数调用 —— 纯音频输出模型

### 二、GPT-4o Transcribe / Mini Transcribe（语音转文本）

#### 模型版本
| 模型标识 | 定位 |
|---------|------|
| `gpt-4o-transcribe` | 最高精度转录 |
| `gpt-4o-mini-transcribe` | 速度/成本优化转录 |

#### 与 Whisper 的对比

被形容为 *"Whisper after it took a UX design course"*（Whisper 上了一门用户体验设计课之后的样子）：

| 特性 | Whisper v2 | GPT-4o Transcribe |
|------|-----------|-------------------|
| **整体准确率** | 高 | **更高，行业领先** |
| **多语言支持** | 好 | **更好，几乎所有测试语言超越** |
| **噪声环境** | 一般 | **显著改善** |
| **口音处理** | 一般 | **显著改善** |
| **文件大小限制** | 较小，需切分大文件 | **更大的文件限制** |
| **字幕生成** | 需额外处理 | **自动 .srt 字幕生成** |

#### 基准表现

- **GPT-4o Transcribe** 在商业 STT 工具中**准确率最高**（基于 WER 和 CER 基准）
- 在清洁音频环境中可达 **95-99%** 准确率
- 对于印度和达罗毗荼语系的挑战性语言（Tamil、Telugu、Malayalam、Kannada），词错误率接近 **30%**（满分 120% 的复合评分标准下）

#### 竞品对比 (2025 WER 基准)

| 模型/提供商 | WER (清洁音频) |
|------------|---------------|
| **GPT-4o Transcribe** | **最优** |
| Gemini 2.5 Pro | ~3.0% |
| Gemini 2.0 Flash | ~4.0% |
| AssemblyAI Universal | ~4.2% |
| Deepgram Nova-3 | ~4.8% |
| NVIDIA Canary Qwen 2.5B | ~5.63% |

#### 多语言和真实世界性能

训练于**多样化、高质量音频数据集**，关键改进：
- 更好的口音和多样化语音处理
- 嘈杂/混乱环境下的性能提升
- 在几乎所有测试语言上超越 Whisper

已知挑战仍然存在：
- 低质量麦克风：准确率下降 15-25%
- 多人重叠说话：准确率下降 25-40%
- 技术术语：准确率下降 15-30%

### 三、技术趋势：从独立模型到语言模型原生能力

这些音频模型延续了 OpenAI 将专业能力集成到通用语言模型中的趋势：
- TTS 不再是独立的音频合成系统，而是**基于 GPT-4o Mini 语言模型**
- Transcribe 不再是独立的 Whisper 系统，而是**基于 GPT-4o 架构**
- 语言模型的上下文理解能力被用来提升音频任务的质量

## 核心洞察

1. **"可提示语音"是语音交互的范式转换**：传统 TTS 系统通过参数调整语音风格（如音高、语速），而 gpt-4o-mini-tts 通过**自然语言 prompt** 控制语音情感和风格。这意味着开发者不再需要理解音频工程参数，只需用日常语言描述想要的效果。这极大降低了语音应用的开发门槛。

2. **Whisper 的退场与"模型即平台"的战略**：GPT-4o Transcribe 全面替代 Whisper，标志着 OpenAI 正在将所有专业能力**统一到 GPT-4o 平台模型**中。独立的专用模型（如 Whisper、DALL-E）逐渐成为"遗留系统"，未来的 AI 能力将通过统一的平台模型提供。

3. **音频领域的竞争格局**：GPT-4o Transcribe 虽然在准确率上领先，但 Google Gemini 系列（2.5 Pro ~3.0% WER）紧随其后。音频 AI 领域正在成为新的竞争前线，尤其在多语言、实时转录、和语音智能体等场景中。

4. **开发者生态的音频化**：这些模型通过标准 API 提供，意味着**语音交互将成为更多应用的默认功能**而非专业功能。以 ~$0.015/分钟 的成本，大规模语音应用在经济上变得可行。

5. **多语言能力的战略意义**：在几乎所有测试语言上超越 Whisper，特别是对南亚和东南亚语言的改进，表明 OpenAI 正在积极扩展其全球市场覆盖。多语言准确率直接影响非英语市场的用户体验和商业机会。
