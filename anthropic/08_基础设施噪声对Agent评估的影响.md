# 基础设施噪声对 Agent 评估的影响

## 文章信息
- **标题**: Quantifying Infrastructure Noise in Agentic Coding Evals
- **发布日期**: 2026-02-03
- **链接**: https://www.anthropic.com/engineering/infrastructure-noise

## 核心内容总结

这篇文章揭示了一个惊人的发现:**仅基础设施配置差异就能导致 Agent 编码基准分数波动高达 6 个百分点——超过了排行榜上顶级模型之间的典型差距**。这对 Agent 评估的可靠性和可比性提出了重大质疑。

### 核心发现

**基础设施影响的量化**:
- **严格执行 (1x 规格)**: 5.8% 的基础设施错误率
- **3x headroom**: 2.1% 的错误率
- **无上限资源**: 0.5% 的错误率

**关键洞察**:
- 超过 3x 阈值后,"额外资源开始积极帮助 Agent 解决它之前无法解决的问题"
- 资源限制从根本上改变了基准测量的内容
- 基础设施配置不仅影响错误率,还影响成功率

### 深层含义

**评估的根本性问题**:
- 不同基础设施配置下的评估结果不可比
- 排行榜上的模型排名可能受基础设施影响
- 需要标准化基础设施配置

**资源与能力的关系**:
- 充足的资源可以让 Agent 尝试更多策略
- 资源限制可能导致 Agent 放弃某些解决方案
- 基础设施成为 Agent 能力的一部分

### 实践建议

**标准化要求**:
- 明确报告基础设施配置
- 使用标准化的评估环境
- 提供充足的资源余量(至少 3x)

**评估设计**:
- 考虑基础设施对评估结果的影响
- 设计对资源不敏感的评估任务
- 或者明确资源限制作为评估的一部分

## 资源影响分析

**资源不足的影响**:
- 增加基础设施错误率
- 限制 Agent 的策略选择
- 导致任务失败或超时
- 影响评估结果的可比性

**充足资源的好处**:
- 降低基础设施错误率
- 允许 Agent 尝试更多策略
- 提高任务成功率
- 改善评估结果的可靠性

**资源与能力的关系**:
- 资源是 Agent 能力的一部分
- 资源限制改变了评估的内容
- 需要明确资源限制的评估目标

## 标准化建议

**评估环境标准化**:
1. 明确 CPU、内存、磁盘规格
2. 提供至少 3x 资源余量
3. 使用容器化隔离环境
4. 监控资源使用情况
5. 报告基础设施配置

**评估报告标准化**:
1. 报告基础设施配置
2. 报告资源使用情况
3. 报告基础设施错误率
4. 提供可重现的配置
5. 说明资源限制的影响

这篇文章揭示了 Agent 评估中一个容易被忽视但影响重大的因素,为提高评估可靠性提供了重要指导。
