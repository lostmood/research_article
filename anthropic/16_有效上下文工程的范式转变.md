# 有效上下文工程: 从 Prompt 到上下文的范式转变

## 文章信息
- **标题**: Effective Context Engineering for AI Agents
- **发布日期**: 2025-09-29
- **链接**: https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents

## 核心内容总结

这篇文章提出从 **"prompt 工程"到"上下文工程"的范式转变**,将上下文工程定义为**在 LLM 推理过程中策划和维护最优 token 集合的策略**。

### 核心概念

**注意力预算 (Attention Budget)**:
- 模型的注意力是有限的资源
- 上下文越大,模型对特定信息的注意力越分散
- 需要精心管理注意力的分配

**上下文腐化 (Context Decay)**:
- 随着更多 token 被添加到上下文窗口,模型对特定关键信息的召回能力会下降
- 无关信息会"污染"上下文
- 需要主动管理和清理上下文

### 指导原则

**核心原则**: 找到"**最小可能的高信号 token 集合**"
- 每个上下文窗口中的 token 都应证明其存在的合理性
- 优先保留高价值信息
- 移除低价值或冗余信息

### 实践策略

**1. 系统 Prompt 设计**:
- 简洁明确的指令
- 避免冗余和重复
- 只包含必要的信息

**2. 按需工具调用**:
- 不预加载所有工具定义
- 动态发现和加载工具
- 减少工具描述的 token 消耗

**3. 压缩与摘要技术**:
- 压缩长文本
- 生成摘要而非全文
- 提取关键信息

**4. 即时信息供给架构**:
- 在需要时才加载信息
- 避免预加载大量信息
- 动态管理上下文内容

## 上下文工程策略

### 1. 系统 Prompt 优化

**优化前**:
```
You are a helpful AI assistant. You should always be polite
and respectful. You should provide accurate and helpful
information. You should follow the user's instructions
carefully. You should ask for clarification if needed...
(200+ tokens)
```

**优化后**:
```
You are a helpful AI assistant. Provide accurate information
and ask for clarification when needed.
(20 tokens)
```

### 2. 工具定义优化

**传统方式**:
```
预加载 100 个工具定义
每个工具 50 tokens
总计: 5000 tokens
```

**按需加载**:
```
使用 Tool Search Tool
只加载需要的 3 个工具
总计: 150 tokens
节省: 97%
```

### 3. 信息压缩

**原始输出**:
```
File: /path/to/file.txt
Size: 1024 bytes
Last Modified: 2026-04-02T10:00:00Z
Content:
[1000 lines of content...]
```

**压缩输出**:
```
File: /path/to/file.txt (1KB, modified 2026-04-02)
Summary: [3-sentence summary]
Key points: [bullet points]
```

### 4. 即时信息供给

**预加载方式**:
```
加载所有可能需要的信息
上下文: 10000 tokens
实际使用: 1000 tokens
浪费: 90%
```

**即时供给**:
```
在需要时才加载信息
上下文: 1000 tokens
实际使用: 1000 tokens
浪费: 0%
```

## 上下文腐化管理

**腐化来源**:
1. 冗余信息
2. 过时信息
3. 低价值信息
4. 重复信息
5. 无关信息

**管理策略**:
1. 定期清理过时信息
2. 压缩历史对话
3. 移除冗余内容
4. 优先保留高价值信息
5. 使用摘要替代全文

## 上下文工程 vs Prompt 工程

| 维度 | Prompt 工程 | 上下文工程 |
|------|-----------|-----------|
| 关注点 | 单次 Prompt | 整个上下文窗口 |
| 范围 | 静态指令 | 动态信息管理 |
| 目标 | 清晰指令 | 最优 token 集合 |
| 方法 | 优化措辞 | 策划和维护 |
| 时间 | 一次性 | 持续过程 |
| 复杂度 | 低 | 高 |
