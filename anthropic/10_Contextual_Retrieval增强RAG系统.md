# Contextual Retrieval: 增强 RAG 系统的检索准确性

## 文章信息
- **标题**: Introducing Contextual Retrieval
- **发布日期**: 2024-09-19
- **链接**: https://www.anthropic.com/engineering/contextual-retrieval

## 核心内容总结

这篇文章介绍了 Anthropic 提出的 **Contextual Retrieval** 技术,通过**在嵌入和索引前为每个文本块添加特定的解释性上下文**来增强 RAG (Retrieval-Augmented Generation) 系统。

### 核心技术

**Contextual Retrieval 方法**:
- 为每个文本块生成解释性上下文
- 将上下文与原文本块一起嵌入和索引
- 检索时使用增强后的文本块

**技术组合**:
1. **Contextual Embeddings**: 为向量嵌入添加上下文
2. **Contextual BM25**: 为关键词检索添加上下文
3. **重排序 (Reranking)**: 对检索结果进行重排序

### 性能提升

**检索失败率降低**:
- Contextual Embeddings + Contextual BM25: **49% 降低**
- 配合重排序: **67% 降低**

**关键价值**:
- 显著改进检索准确性
- 直接转化为下游任务的更好表现
- 提升 RAG 系统的整体质量

### 技术原理

**问题**: 传统 RAG 系统中,文本块脱离上下文后难以理解

**解决方案**: 为每个文本块添加解释性上下文,说明:
- 这个文本块来自哪里
- 它讨论的是什么主题
- 它与文档的关系

**实现**: 使用 LLM 为每个文本块生成上下文描述

## Contextual Retrieval 工作流程

```
1. 文档分块
   ↓
2. 为每个块生成上下文描述 (使用 LLM)
   ↓
3. 将上下文与原文本块合并
   ↓
4. 生成向量嵌入 (Contextual Embeddings)
   ↓
5. 建立 BM25 索引 (Contextual BM25)
   ↓
6. 检索时使用增强后的文本块
   ↓
7. 可选: 重排序检索结果
   ↓
8. 返回最相关的文本块
```

## 上下文生成示例

**原文本块**:
```
The company's revenue increased by 25% in Q3.
```

**生成的上下文**:
```
This sentence is from the Q3 2024 financial report,
discussing the company's revenue performance in the
third quarter compared to the previous quarter.
```

**增强后的文本块**:
```
[Context: This sentence is from the Q3 2024 financial
report, discussing the company's revenue performance
in the third quarter compared to the previous quarter.]

The company's revenue increased by 25% in Q3.
```

## 技术细节

**上下文生成策略**:
- 使用 LLM 理解文本块的含义
- 生成简洁的上下文描述(1-2 句话)
- 包含文档来源、主题、关系等信息
- 避免冗余和重复

**检索策略**:
- 同时使用向量检索和关键词检索
- 组合两种方法的结果
- 使用重排序模型进一步优化
- 返回最相关的 top-k 结果

**性能优化**:
- 批量生成上下文描述
- 缓存常用的上下文
- 优化向量数据库查询
- 并行化检索操作
