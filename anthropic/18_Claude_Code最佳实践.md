## Claude Code 最佳实践

### 1. 项目配置

**CLAUDE.md 示例**:
```markdown
# 项目: My App

## 技术栈
- Language: Python 3.11
- Framework: FastAPI
- Database: PostgreSQL
- Testing: pytest

## 代码风格
- Follow PEP 8
- Use type hints
- Write docstrings for public functions

## 常用命令
- Run tests: `pytest tests/`
- Start server: `uvicorn main:app --reload`
- Format code: `black .`

## 项目结构
- `src/`: Source code
- `tests/`: Test files
- `docs/`: Documentation
```

### 2. Prompt 策略

**清晰描述任务**:
```
❌ 不好: "修复这个 bug"
✅ 好: "修复用户登录时密码验证失败的 bug,
      错误信息显示在 logs/error.log 中"
```

**提供上下文**:
```
❌ 不好: "添加一个新功能"
✅ 好: "在用户管理模块添加批量导入用户的功能,
      支持 CSV 格式,参考现有的单个用户创建逻辑"
```

**分解复杂任务**:
```
❌ 不好: "重构整个认证系统"
✅ 好: "第一步:将密码哈希逻辑提取到独立模块
      第二步:添加 JWT token 支持
      第三步:实现刷新 token 机制"
```

### 3. 工具使用模式

**文件编辑**:
- 使用 Edit Tool 进行精确修改
- 避免重写整个文件
- 保持代码风格一致

**命令执行**:
- 使用 Bash 执行测试
- 运行代码格式化工具
- 执行构建和部署命令

**代码搜索**:
- 使用 Grep 搜索代码
- 使用 Glob 查找文件
- 快速定位相关代码

### 4. 工作流优化

**迭代式开发**:
```
1. 理解需求
   ↓
2. 设计方案
   ↓
3. 实现代码
   ↓
4. 运行测试
   ↓
5. 修复问题
   ↓
6. 重复 3-5 直到完成
```

**测试驱动**:
```
1. 编写测试用例
   ↓
2. 运行测试(应该失败)
   ↓
3. 实现功能
   ↓
4. 运行测试(应该通过)
   ↓
5. 重构代码
```

### 5. 代码库导航

**大型代码库策略**:
1. 从入口文件开始
2. 理解项目结构
3. 查找相关模块
4. 阅读关键代码
5. 理解数据流

**高效搜索**:
- 搜索函数定义
- 搜索类定义
- 搜索使用示例
- 搜索测试用例

### 6. 测试策略

**测试类型**:
- 单元测试: 测试单个函数
- 集成测试: 测试模块交互
- 端到端测试: 测试完整流程

**测试实践**:
- 及时运行测试
