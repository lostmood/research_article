# MCP 代码执行: 构建更高效的 Agent

## 文章信息
- **标题**: Code Execution with MCP: Building More Efficient Agents
- **发布日期**: 2025-11-04
- **链接**: https://www.anthropic.com/engineering/code-execution-with-mcp

## 核心内容总结

这篇文章探讨 Agent 通过**编写代码调用工具**来扩展其能力的方式。展示了如何利用 **MCP (Model Context Protocol)** 实现代码执行,使 Agent 能够以更高效、更灵活的方式与外部工具和服务交互,**减少 Agent 与工具之间的往返次数**。

### 核心概念

**代码执行的优势**:
- 批量操作: 通过循环批量调用工具
- 复杂逻辑: 支持条件判断和控制流
- 中间结果: 将中间结果保留在代码执行环境中
- 减少往返: 一次代码执行完成多次工具调用

**MCP 的作用**:
- 提供标准化的协议
- 支持代码执行能力
- 统一的工具调用接口
- 跨平台和跨语言支持

### 使用场景

**适合代码执行的场景**:
1. 需要批量处理数据
2. 工具调用有复杂逻辑
3. 需要循环或条件判断
4. 中间结果多且大
5. 需要减少往返次数

**代码执行示例**:
```python
# 批量处理文件
results = []
for file in files:
    content = read_file(file)
    if "error" in content:
        result = analyze_error(content)
        results.append(result)
return results
```

## 代码执行模式

### 批量处理模式

```python
# 批量读取和处理文件
results = []
for file_path in file_paths:
    content = read_file(file_path)
    processed = process_content(content)
    results.append(processed)
return results
```

**优势**:
- 一次代码执行完成多次工具调用
- 中间结果不占用上下文
- 减少往返次数

### 条件处理模式

```python
# 根据条件选择不同的处理方式
for item in items:
    if item.type == "A":
        result = process_a(item)
    elif item.type == "B":
        result = process_b(item)
    else:
        result = process_default(item)
    save_result(result)
```

**优势**:
- 支持复杂的条件逻辑
- 避免多次往返判断
- 提高处理效率

### 聚合处理模式

```python
# 聚合多个数据源的结果
data = []
for source in sources:
    result = fetch_data(source)
    data.extend(result)

# 在代码中进行聚合和分析
summary = analyze_data(data)
return summary
```

**优势**:
- 在代码中聚合数据
- 减少数据传输
- 优化 token 使用

### 错误处理模式

```python
# 带错误处理的批量操作
results = []
errors = []
for item in items:
    try:
        result = process_item(item)
        results.append(result)
    except Exception as e:
        errors.append({"item": item, "error": str(e)})

return {"results": results, "errors": errors}
```

**优势**:
- 完善的错误处理
- 部分失败不影响整体
- 提供详细的错误信息

```

  ↓

```

## 与传统方式的对比

**传统工具调用**:
```
Agent: 调用 tool1(item1)
  ↓ 往返
Tool: 返回 result1
  ↓ 往返
Agent: 调用 tool1(item2)
  ↓ 往返
Tool: 返回 result2
  ↓ 往返
...
总计: N 次往返
```

**代码执行**:
```
Agent: 执行代码
  ↓ 1次往返
Code:
  for item in items:
    result = tool1(item)
    results.append(result)
  return results
  ↓ 1次往返
Agent: 收到所有结果
总计: 1 次往返
```
