# Claude Code Auto Mode: 更安全的权限跳过方式

## 文章信息
- **标题**: Claude Code Auto Mode: A Safer Way to Skip Permissions
- **发布日期**: 2026-03-25
- **链接**: https://www.anthropic.com/engineering/claude-code-auto-mode

## 核心内容总结

这篇文章介绍了 Anthropic 为 Claude Code 推出的 **auto mode**,一个**两层权限系统**,用**基于模型的分类器替代手动审批提示**。系统使用**快速单 token 过滤器加思维链推理**来评估工具调用的安全性,实现 **0.4% 的误报率**,同时能捕获凭据探测和权限升级等危险操作。

### 核心设计

**两层权限系统**:
1. **第一层**: 快速单 token 过滤器
   - 快速判断明显安全或危险的操作
   - 低延迟,高吞吐量
   - 处理大部分常规操作

2. **第二层**: 思维链推理
   - 对不确定的操作进行深度分析
   - 考虑上下文和意图
   - 提供详细的安全评估

### 性能指标

**准确性**:
- **误报率**: 0.4% (安全操作被错误拒绝)
- **漏报率**: 17% (危险操作未被检测)
- **捕获能力**: 能捕获凭据探测、权限升级等危险操作

**权衡**:
- 允许常规项目内编辑无需审查
- 对外部操作进行分类器评估
- 在用户意图模糊的情况下存在漏报

### 工作原理

**分类流程**:
```
工具调用请求
  ↓
快速过滤器 (单 token)
  ├─→ 明显安全 → 自动允许
  ├─→ 明显危险 → 自动拒绝
  └─→ 不确定 → 思维链推理
        ↓
      深度分析
        ├─→ 安全 → 允许
        └─→ 危险 → 拒绝/提示
```

**考虑因素**:
- 操作类型(读/写/执行/删除)
- 操作范围(项目内/项目外/系统)
- 文件路径和内容
- 用户意图和上下文
- 历史操作模式

## 思维链推理示例

**操作**: `bash("rm -rf /tmp/cache")`

**推理过程**:
```
1. 操作分析:
   - 命令: rm -rf
   - 目标: /tmp/cache
   - 类型: 删除操作

2. 上下文分析:
   - 用户意图: 清理临时缓存
   - 历史操作: 之前创建了 /tmp/cache
   - 项目需求: 清理构建缓存

3. 风险评估:
   - 路径: /tmp 是临时目录,相对安全
   - 范围: 仅限 cache 子目录
   - 影响: 可恢复,不影响系统

4. 决策:
   - 结论: 安全
   - 理由: 清理临时缓存是合理操作
   - 行动: 允许执行
```

## 安全策略配置

**默认策略**:
```yaml
auto_mode:
  enabled: true
  project_operations:
    read: auto_allow
    write: auto_allow
    execute: auto_allow
  external_operations:
    read: classifier
    write: classifier
    execute: classifier
  dangerous_operations:
    delete: classifier
    system: deny
```

**自定义策略**:
```yaml
custom_rules:
  - pattern: "/home/user/data/*"
    operations: [read]
    action: auto_allow
  - pattern: "*.sh"
    operations: [execute]
    action: classifier
  - pattern: "/etc/*"
    operations: [read, write]
    action: deny
```
