# 为 Agent 编写有效工具的方法论

## 文章信息
- **标题**: Writing Effective Tools for Agents — with Agents
- **发布日期**: 2025-09-11
- **链接**: https://www.anthropic.com/engineering/writing-tools-for-agents

## 核心内容总结

这篇文章聚焦于一个核心观点:**AI Agent 的效果取决于其工具质量**,提出了**以 Agent 为中心的工具设计方法论**。

### 核心方法论

**三步迭代流程**:
1. **原型制作 (Prototype)**: 快速创建工具原型
2. **评估 (Evaluate)**: 测试工具在实际场景中的表现
3. **协作 (Collaborate)**: 利用 Claude Code 分析失败并自动优化工具

**关键洞察**:
- 使用 Agent 来帮助优化 Agent 的工具
- 迭代改进比一次性完美设计更有效
- 实际使用中的失败是最好的反馈

### 五个关键设计原则

1. **选择合适的工具 (Choose the Right Tools)**
   - 提供必要的工具,避免冗余
   - 工具应该匹配任务需求
   - 考虑工具之间的组合

2. **命名空间隔离 (Namespace Isolation)**
   - 使用清晰的命名空间避免冲突
   - 特别重要在多 Agent 场景
   - 提高工具的可维护性

3. **返回有意义的上下文 (Return Meaningful Context)**
   - 工具输出应该包含足够的上下文信息
   - 帮助 Agent 理解结果
   - 减少后续的澄清请求

4. **优化 Token 效率 (Optimize Token Efficiency)**
   - 减少不必要的输出
   - 使用简洁的格式
   - 考虑上下文窗口的限制

5. **像编写 Prompt 一样设计工具描述 (Write Tool Descriptions Like Prompts)**
   - 工具描述是 Agent 理解工具的关键
   - 提供清晰的使用示例
   - 说明参数的含义和约束

### 实践方法

**使用 Claude Code 优化工具**:
- 让 Claude Code 分析工具使用失败
- 自动识别工具设计问题
- 生成改进建议和代码

**迭代优化循环**:
1. 创建工具原型
2. 在实际任务中测试
3. 收集失败案例

6. 重复循环

## 五个设计原则详解

### 1. 选择合适的工具

**考虑因素**:
- 任务需求: 工具应该匹配任务的需求
- 工具数量: 避免提供过多或过少的工具
- 工具组合: 考虑工具之间的组合使用
- 功能覆盖: 确保必要的功能被覆盖

**反面案例**:
- 提供过多冗余的工具
- 缺少关键功能的工具
- 工具功能重叠导致混淆

### 2. 命名空间隔离

**重要性**:
- 避免工具名称冲突
- 提高代码可维护性
- 特别重要在多 Agent 场景
- 清晰的组织结构

**实践方法**:
```
filesystem:read
filesystem:write
database:query
database:update
network:http_get
network:http_post
```

### 3. 返回有意义的上下文

**包含的信息**:
- 操作结果: 成功或失败
- 相关数据: 返回的数据或状态
- 错误信息: 清晰的错误描述
- 上下文信息: 帮助理解结果的信息

**示例**:
```json
{
  "success": true,
  "file_path": "/path/to/file.txt",
  "content": "...",
  "size": 1024,
  "last_modified": "2026-04-02T10:00:00Z"
}
```

### 4. 优化 Token 效率

**优化策略**:
- 减少冗余输出
- 使用简洁的格式
- 分页或截断大量数据
- 提供摘要而非全文

**权衡**:
- Token 效率 vs 信息完整性
- 需要找到平衡点
- 根据使用场景调整

### 5. 工具描述像 Prompt

**描述要素**:
- 清晰的功能说明
- 参数的含义和约束
- 使用示例
- 返回值说明
- 错误情况说明

**示例**:
```
Tool: read_file
Description: Read the contents of a file from the filesystem.
Parameters:
  - file_path (string, required): The absolute path to the file
  - encoding (string, optional): File encoding, defaults to 'utf-8'
Returns: File contents as a string
Errors: FileNotFoundError if file doesn't exist
Example: read_file("/path/to/file.txt")
```

## 使用 Claude Code 优化工具

**工作流程**:
1. 提供工具代码和失败案例给 Claude Code
2. Claude Code 分析失败原因
3. Claude Code 识别工具设计问题
4. Claude Code 生成改进建议
5. Claude Code 可以直接修改工具代码
6. 重新测试改进后的工具

**优势**:
- 自动化分析和优化
- 快速迭代
- 发现人类可能忽视的问题
- 生成高质量的改进代码

## 常见工具设计问题

**问题 1: 工具描述不清晰**
- 症状: Agent 不知道何时使用工具
- 解决: 提供清晰的功能说明和使用示例

**问题 2: 返回信息不足**
- 症状: Agent 需要多次调用工具获取信息
- 解决: 返回有意义的上下文信息

**问题 3: Token 消耗过大**
- 症状: 工具输出占用大量上下文
- 解决: 优化输出格式,提供摘要

**问题 4: 错误处理不当**
- 症状: 工具失败时 Agent 不知道如何处理
- 解决: 返回清晰的错误信息和建议

**问题 5: 工具功能重叠**
- 症状: Agent 不知道选择哪个工具
- 解决: 明确工具的职责边界
