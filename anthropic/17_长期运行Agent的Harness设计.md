# 长期运行 Agent 的 Harness 设计

## 文章信息
- **标题**: Effective Harnesses for Long-Running Agents
- **发布日期**: 2025-11-26
- **链接**: https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents

## 核心内容总结

这篇文章探讨 Agent **跨多个上下文窗口持续工作**的改进方法。介绍了长时运行 Agent 的 **harness (线束)** 设计模式,解决 Agent 在长期任务中面临的**上下文管理、状态持久化和错误恢复**等核心挑战。

### 核心挑战

**长期运行的问题**:
1. **上下文窗口限制**: 单个上下文窗口无法容纳所有信息
2. **上下文腐化**: 长期运行导致上下文质量下降
3. **状态管理**: 需要跨多个会话管理状态
4. **错误恢复**: 需要从失败中恢复并继续
5. **进度跟踪**: 需要跟踪长期任务的进度

### Harness 设计模式

**Harness 的作用**:
- 管理 Agent 的生命周期
- 协调多个上下文窗口
- 持久化状态和进度
- 处理错误和恢复
- 提供检查点机制

**关键组件**:
1. **状态管理器**: 管理 Agent 状态
2. **上下文管理器**: 管理上下文窗口
3. **检查点系统**: 保存和恢复进度
4. **错误处理器**: 处理错误和异常
5. **进度跟踪器**: 跟踪任务进度

### 实践策略

**1. 检查点机制**:
- 定期保存 Agent 状态
- 支持从检查点恢复
- 避免重复工作

**2. 状态持久化**:
- 将状态保存到持久存储
- 支持跨会话恢复
- 管理状态版本

**3. 上下文压缩**:
- 压缩历史信息
- 保留关键信息
- 定期清理冗余

**4. 错误恢复**:
- 捕获和记录错误
- 自动重试机制
- 回退到检查点

```json

  "task_state": {

    "pending_steps": ["testing", "deployment"]
```

## 上下文管理策略

**上下文压缩**:
```
原始上下文: 50000 tokens
  ↓ 压缩历史对话
压缩后: 30000 tokens
  ↓ 移除冗余信息
优化后: 20000 tokens
  ↓ 提取关键信息
最终: 10000 tokens
```

**上下文切换**:
```
阶段 1: 规划
  上下文: 需求 + 历史决策
  ↓ 保存规划结果
阶段 2: 生成
  上下文: 规划结果 + 代码库
  ↓ 保存生成结果
阶段 3: 评估
  上下文: 生成结果 + 测试结果
```

## 错误恢复机制

**错误类型**:
1. **工具错误**: 工具调用失败
2. **上下文溢出**: 上下文窗口满
3. **基础设施错误**: 网络、资源等问题
4. **逻辑错误**: Agent 逻辑错误
5. **用户中断**: 用户主动中断

**恢复策略**:
```
错误发生
  ↓
捕获和记录错误
  ↓
判断错误类型
  ↓
选择恢复策略
  ├── 自动重试
  ├── 回退到检查点
  ├── 请求用户干预
  └── 终止任务
```

## 与短期任务的对比

| 维度 | 短期任务 | 长期任务 |
|------|---------|---------|
| 时长 | 分钟级 | 小时/天级 |
| 上下文 | 单个窗口 | 多个窗口 |
| 状态 | 内存中 | 持久化 |
| 错误恢复 | 简单重试 | 检查点恢复 |

| 复杂度 | 低 | 高 |
