# Claude Code 沙箱: 超越权限提示的安全性

## 文章信息
- **标题**: Beyond Permission Prompts: Making Claude Code More Secure
- **发布日期**: 2025-10-20
- **链接**: https://www.anthropic.com/engineering/claude-code-sandboxing

## 核心内容总结

这篇文章介绍了 Claude Code 的**沙箱功能**,通过**减少权限提示来提升用户安全性**。该方案在保持 Agent 自主操作能力的同时,通过**隔离执行环境**来限制潜在的安全风险,平衡了开发效率和安全防护需求。

### 核心概念

**沙箱的作用**:
- 隔离执行环境
- 限制文件系统访问
- 控制网络访问
- 防止恶意操作
- 保护主系统

**设计目标**:
- 减少权限提示的频率
- 提升用户安全性
- 保持开发效率
- 平衡安全和便利

### 沙箱特性

**隔离机制**:
1. **文件系统隔离**: 限制可访问的文件和目录
2. **网络隔离**: 控制网络访问权限
3. **进程隔离**: 隔离进程执行环境
4. **资源限制**: 限制 CPU、内存等资源使用

**安全边界**:
- 项目内操作: 自动允许
- 项目外操作: 需要权限审查
- 系统操作: 严格限制
- 网络操作: 根据策略控制

### 用户体验

**减少提示**:
- 项目内的常规操作无需提示
- 危险操作仍需确认
- 提供清晰的安全边界
- 用户可以配置策略

**安全保障**:
- 防止意外的文件删除
- 防止访问敏感文件
- 防止恶意网络请求
- 记录所有操作日志

## 安全边界

**项目内操作** (自动允许):
- 读取项目文件
- 修改项目文件
- 创建项目文件
- 运行项目命令

**项目外操作** (需要审查):
- 访问项目外文件
- 修改系统文件
- 网络请求
- 系统命令

**危险操作** (严格限制):
- 删除重要文件
- 修改系统配置
- 访问敏感数据
- 执行特权命令

## 隔离机制

### 文件系统隔离

**允许访问**:
```
/project/
  ├── src/
  ├── tests/
  ├── docs/
  └── ...
```

**禁止访问**:
```
/home/user/.ssh/
/home/user/.aws/
/etc/
/sys/
```

### 网络隔离

**允许**:
- 访问公开 API
- 下载依赖包
- 访问文档网站

**禁止**:
- 访问内网地址
- 上传敏感数据
- 未授权的网络请求

### 进程隔离

**隔离特性**:
- 独立的进程空间
- 限制进程权限
- 监控进程行为
- 防止进程逃逸

### 资源限制

**限制项**:
- CPU 使用率
- 内存使用量
- 磁盘空间
- 网络带宽
- 进程数量

## 权限策略

**默认策略**:
```yaml
project_operations:
  read: allow
  write: allow
  execute: allow

system_operations:
  read: prompt
  write: deny
  execute: deny

network_operations:
  public_api: allow
  internal: deny
  upload: prompt
```

**自定义策略**:
```yaml
custom_rules:
  - path: "/home/user/data"
    operations: [read]
    action: allow
  - path: "/etc"
    operations: [read, write]
    action: deny
  - network: "api.example.com"
    action: allow
```

## 常见安全场景

**场景 1: 读取敏感文件**
- 检测: 沙箱检测到访问 ~/.ssh/
- 处理: 阻止访问,提示用户
- 日志: 记录尝试访问敏感文件

**场景 2: 删除重要文件**
- 检测: 沙箱检测到删除操作
- 处理: 提示用户确认
- 日志: 记录删除操作

**场景 3: 网络请求**
- 检测: 沙箱检测到网络请求
- 处理: 根据策略允许或拒绝
- 日志: 记录网络请求

**场景 4: 执行系统命令**
- 检测: 沙箱检测到系统命令
- 处理: 在隔离环境中执行
- 日志: 记录命令执行
