# Think Tool: 为 Claude 提供专用推理空间

## 文章信息
- **标题**: The "think" Tool: Enabling Claude to Stop and Think
- **发布日期**: 2025-03-20
- **链接**: https://www.anthropic.com/engineering/claude-think-tool

## 核心内容总结

这篇文章介绍了 Anthropic 推出的 **"think" 工具**,为 Claude 在复杂多步骤任务中提供**专用推理空间**,区别于在响应生成前发生的 extended thinking。

### 核心概念

**Think Tool 的作用**:
- 提供专用的推理空间
- 区别于 extended thinking (扩展思考)
- 在工具调用过程中进行推理
- 推理内容不直接返回给用户

**与 Extended Thinking 的区别**:
- **Extended Thinking**: 响应生成前的内部思考
- **Think Tool**: 工具调用过程中的显式推理
- **时机**: Think Tool 可以在任务执行过程中多次使用
- **可见性**: Think Tool 的推理过程可以被记录和分析

### 性能提升

**τ-Bench 基准**:
- 航空领域配合优化提示: **54% 的相对改进**
- 展示了在特定领域的显著提升

**SWE-bench**:
- 平均提升 **1.6%**
- 贡献了 SOTA 结果

**最有效的场景**:
1. 涉及工具输出分析的任务
2. 策略密集型环境
3. 高代价顺序决策场景

### 实现特点

**简单实现**:
- 代码开销极小
- 易于集成到现有系统
- 不需要复杂的架构改动

**使用模式**:
- Agent 可以主动调用 think tool
- 在需要推理时使用
- 推理结果指导后续行动

## Think Tool 使用场景

**最有效的场景**:
1. **工具输出分析**: 分析工具返回的复杂结果
2. **策略密集型任务**: 需要制定复杂策略的任务
3. **高代价决策**: 决策错误代价高的场景
4. **多步骤任务**: 需要规划多个步骤的任务
5. **问题诊断**: 诊断复杂问题的根源
6. **代码推理**: 推理代码的逻辑和结构

**不太需要的场景**:
1. 简单的单步骤任务
2. 明确的操作指令
3. 不需要策略的任务
4. 低复杂度的决策

## Think Tool vs Extended Thinking

| 维度 | Think Tool | Extended Thinking |
|------|-----------|------------------|
| 时机 | 任务执行过程中 | 响应生成前 |
| 次数 | 可多次使用 | 每次响应一次 |
| 可见性 | 可记录和分析 | 内部过程 |
| 控制 | Agent 主动调用 | 自动触发 |
| 用途 | 工具输出分析、策略制定 | 整体响应规划 |
| 上下文 | 在上下文窗口中 | 在上下文窗口外 |

## 推理模式

**分析型推理**:
- 分析工具输出
- 理解任务状态
- 识别问题和机会

**策略型推理**:
- 制定行动计划
- 评估不同选项
- 选择最佳策略

**诊断型推理**:
- 诊断问题根源
- 分析错误原因
- 制定解决方案

**预测型推理**:
- 预测行动结果
- 评估风险和收益
- 调整策略

**集成 Think Tool**:
1. 定义 think tool 的接口
2. 在 Agent 系统中注册工具
3. 训练 Agent 何时使用 think tool
4. 记录和分析推理过程
5. 根据效果优化使用策略

**优化使用**:
1. 在复杂决策前使用
2. 分析工具输出时使用
3. 制定策略时使用
4. 诊断问题时使用
